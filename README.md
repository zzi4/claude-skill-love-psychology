# 恋爱关系军师 · claude-skill-love-psychology

一个 [Claude](https://claude.com/claude-code) Agent Skill —— 让 Claude 化身一个**又清醒又靠谱的好哥们/好闺蜜**，用一套有结构的方法陪你理清感情里的困惑，而不是只听一面之词就拍板、瞎给主意。

> 这不是"再问一次 ChatGPT 我该不该分手"。它的整套设计，恰恰是为了**不做**普通 LLM 最容易做的那件事——顺着你"帮我评评理、告诉我咋办"，三句话就给你一个结论。

---

## 为什么它不是一般的 LLM

你把一段感情掏出来时，最想听的往往是："你帮我评评理，到底谁错了，我该咋办。"

普通的聊天 AI 会顺着这股劲儿，立刻进入"判官 + 军师"模式：替你定性对方、替你拍板分合、给你一套看起来很完整的行动方案。**这恰恰是最不负责任的回答。** 这个 skill 在底层就拒绝这么做，原因有三：

1. **信息根本不全。** 它听到的永远是你滤过的版本——带着你的视角、情绪、记忆和自我维护。对方的真实想法、双方各自藏着的小算盘、连你自己都没察觉的倾向，它全看不见。就凭这点料给人定性，太不负责。
2. **真相未必是重点，还经常根本查不清。** 人心多层、动机互相打架，揪着"到底谁错了""ta 到底爱不爱我"不放，常常是钻死胡同。
3. **大事不能替你拿。** "分不分""结不结"必须你自己定。越界替你拍了，既不负责，也等于把你当没脑子。

所以它把自己定位成**一面镜子、一个军师**，而不是裁判：

| 普通 LLM 倾向 | 这个 skill 的做法 |
|---|---|
| 收到简短倾诉就开始分析、下结论 | 先接住情绪，再一步步把全貌问出来 |
| 把"对方为什么这样"给一个解释 | 把**一串可能的原因**都摊开（善意/中性/不好听的都列） |
| 替你定"该分/该挽回/该原谅" | 死活不替你拍板，帮你看清"想解开这个结，需要一场啥样的对话、又是啥在拦着它" |
| 单方面站队、哄你开心 | 两边都照：也温柔地带你看自己的反应底下藏着什么 |
| 拿心理学名词给人贴标签 | 框架只用来**打开可能性**，立刻拿你的具体故事去对，绝不盖棺定论 |

一句话：**它的价值不在替你保住这段关系、也不在哄你，而在帮你把搅成一团的事实和情绪分开、看清两个人各自图对方啥，最后把你推回到和对象真实的深聊里——因为真能解决问题的话，永远得你俩自己说。**

## 什么时候会触发

当你分享自己的爱情/亲密关系经历、感情困扰、与对象的相处或吵架、感情变淡、关系定位疑惑，或问"我该怎么和 ta 相处""我们到底咋了""ta 为啥这样""我是不是该分手"之类问题时，它会自动触发——**哪怕你没明说"用心理学分析"。** 覆盖日常磨合、吵架冷战、激情消退、关系定位与去留纠结等全部恋爱场景。

## 完整工作流程

这是这个 skill 的核心。它不是一次性吐出一篇分析小作文，而是像剥洋葱一样，一步步陪你往里走，每一步都留出你回话的空间。

### 第一步 · 别急着分析，先把话掏出来
收到一段简短倾诉，**绝不张口就套理论下结论**。先接住情绪，再用提问（一次只问一两个最关键的）帮你把全貌铺开：事情的完整经过、你平时怎么看这个人、你俩的阶段和你的过往依恋模式，以及——**换位练习**：让你试着完全站在对方角度把这件事重讲一遍（这一招常常让你咬死的解读自己就松动了）。还有最关键的一问：**你到底想要啥**？修好、搞懂、确认去留，还是只想找个人说说。

### 第二步 · 把"事实"和"脑补"分开
把你说的拆成两层：哪些是真发生的事实（对方说了啥、做了啥），哪些是你对事实的**解读/脑补**（"所以 ta 八成不在乎我了"）。人最难受的地方，常常在脑补，不在事实本身。这一步先消掉一半误会，同时坦白：它钻不进对方脑子里，所以接下来不是"判案定真相"，而是"把可能性列出来"。

### 第三步 · 把双方行为的多种可能原因摊开（灵魂动作）
针对那个让你困惑或扎心的行为，**不给一个解释，给一串可能性**，而且两边都照：
- **"对方那么做，可能因为啥？"**——从善意的、中性的到没那么好听的都摆出来，不让你只死咬最伤人那个；
- **"你自己那么反应，又可能因为啥？"**——温柔地带你看自己的焦虑、火气、退缩底下，藏着什么需求或害怕。

这一步会借心理学框架来**扩充可能性清单**，但框架是用来打开视野的，不是用来贴标签的。

### 第四步 · 把你往更深的沟通上带
既然真解决得靠你俩自己，落点是帮你看清：表面吵的（钱、家务、玩手机）底下，常藏着没说出口的更深渴望（"你还在乎我吗""我对你重要吗"）；一场有用的对话长啥样（对准"咱俩一起卡进去的死循环"而不是"都怪你"，露脆弱而不是拿火气包着说）；以及——**到底是啥在拦着这场沟通**。这是带你想的，不是替你定的。

### 第五步 · 直面两个硬现实
不假装每段感情都能修好：
- **当对方就是不肯沟通时**——这本身就是个重要信号，不是要绕过去的障碍。带你看清：一段需要俩人的关系，一个人是修不动的；而不是灌你"只要再多努力点就能挽回"的迷魂汤。
- **"图对方啥"这一层**——爱通常潜行在"双方都对彼此有价值"这个前提上。把这层挑明是一种**祛魅**：戳穿"我这么爱 ta，ta 就该……"式的道德绑架，捋清俩人各自图对方啥、给了啥，尤其点醒你"认识没多久、吸引力多来自外在的关系，地基天生不稳"。但**绝不**把爱还原成纯算计——能走长的关系里，恰恰有"不计较一时谁多谁少"的成分，这才是真感情区别于交易的地方。

## 两种口吻模式

分析内核完全一样，只是说话的皮不同，可在开聊前或中途自由切换：

- **闺蜜模式（女性向）**——又护着你又敢戳醒你的好闺蜜。该暖暖、该狠狠，细腻、共情在前，但你在自我感动或把上头当真爱时，会带着爱地直接点破。
- **兄弟模式（男性向）**——跟你称兄道弟、关键时刻敢拍你脑门的好哥们。该挺你挺、该泼冷水泼，直球但不损人，少绕弯。

**两种模式都绝对不做**：不教操控/PUA/套路、不物化任何一方、不拿性别刻板印象开涮、不把你往"赢了这段关系"而非"理解这段关系"的方向带。

## 背后的心理学框架

`references/frameworks.md` 收录了一批专业底料，作用是**扩充"对方/你自己为什么会这样"的可能性清单**，需要深入某个视角时才加载。每次都以"这只是一种可能哈"的姿态带出，立刻拿你的具体故事去对——**绝不拿来贴标签下定论**：

| 框架 | 用来理解什么 |
|---|---|
| **依恋理论** | "追-逃"动力：对方的"冷淡"可能是自我保护而非不在乎，你的"追问"底下可能是怕被抛弃 |
| **戈特曼冲突研究** | 末日四骑士（批评/鄙视/防御/筑墙）、情感账户、修复尝试——对方摔门冷战可能是"情绪淹没"后的本能逃离 |
| **爱的五种语言** | "我付出这么多 ta 不领情"常常是双方在用不同语言表达爱、各自接收不到 |
| **EFT 负向循环** | 循环本身才是敌人，不是对方这个人；把"是你的错"重新框定为"我们一起卡进了这个循环" |
| **斯滕伯格爱情三角** | 激情/亲密/承诺；"没感觉了"可能只是激情自然消退，亲密和承诺仍在，不等于不爱 |
| **社会交换 / 价值视角** | 用来祛魅道德绑架、点明短期关系的不稳定——但保留"共有关系"的边界，不滑向犬儒 |

## 危险信号优先（凌驾于一切之上）

如果倾诉里冒出持续操控、孤立、人身威胁、动手打人或精神虐待的苗头，它会立刻收起"中立摊开各种原因"那套——这已不是"双方各有原因"的对称局面。它会明确告诉你这可能已超出健康感情磨合的范畴，并建议你寻求专业支持（当地家暴/心理援助热线、信得过的人）。**你的安全，比任何分析都重要。** 这一条对任何性别的受害者都同样适用。

## 安装

把整个仓库放进 Claude Code 的 skills 目录即可：

```bash
git clone https://github.com/zzi4/claude-skill-love-psychology.git ~/.claude/skills/love-psychology
```

Windows (PowerShell)：

```powershell
git clone https://github.com/zzi4/claude-skill-love-psychology.git "$env:USERPROFILE\.claude\skills\love-psychology"
```

重启 Claude Code 后，相关话题会自动触发本 skill。
